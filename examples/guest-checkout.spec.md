# Guest checkout — spec

> Canonical (visual review surface): [`./guest-checkout.html`](./guest-checkout.html)
> Status: **approved** · generated by `loupe spec`

## Problem

Forced login before checkout kills guest conversions. Shoppers who do not want an account abandon
the cart at the login wall. Let them pay without creating one.

## Product Lens

- Use cases touched (blast radius): the checkout flow — specifically the **Login required** step (now
  optional) and the path from cart to payment. Browse and Add to cart are unaffected.
- UX change (before → after): `Add to cart → Login required → Pay` becomes `Add to cart → Checkout as
  guest → Pay`, with an optional "save as account" offered **after** the order is confirmed.
- Decisions settled in the grill: account creation is offered after payment, never required; a guest
  order is identified by email + order number.

## Code Lens

- Architecture change (before → after): the `Auth gate (forces login)` in front of the checkout
  service is replaced by a `Guest session` path; checkout no longer depends on an authenticated user.
- Interface delta — added / changed / removed:
  - added: `GuestSession` (email-keyed, no account), `POST /checkout/guest`
  - changed: `Checkout` accepts a guest session as well as an authenticated user
  - removed: the hard auth check between cart and payment
- Grounded in: real codebase (checkout service + auth gate modules).

## Open questions / follow-ups

- Rate-limit guest checkouts by email/IP to limit abuse.
- Decide retention for guest orders that are never converted to accounts.
